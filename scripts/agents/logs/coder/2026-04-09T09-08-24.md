## Summary
- Fixed issue #382: the `effort` transform in `createOpenAiCompatibleConfiguration` gated on an exact `"gpt-5"` model match, so `reasoning_effort` was silently dropped for every registered reasoning shorthand (`gpt-5.2`, `gpt-5-mini`, `gpt-5-nano`, `o3`, `o4-mini`).
- Replaced the allow-list with a prefix match (`gpt-5*`) plus a regex (`^o\d`) for the o-series, keeping the value validation (`minimal|low|medium|high`) intact.
- Hardened the transform against non-string `model` inputs.
- Extended unit tests in `compatible.test.ts` and `openai.test.ts` to cover every registered reasoning shorthand and to confirm non-reasoning models (`gpt-4o`, `gpt-4.1`) still return `undefined`.
- Full suite green: 162 suites / 1599 tests passing, typecheck clean, eslint clean on touched files.
